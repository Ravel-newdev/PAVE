## 📋 Requisitos Funcionais

| ID   | Nome                              | Descrição                                                                                                                                               |
|------|-----------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|
| RF01 | Cadastro de usuário               | O sistema deve permitir o cadastro de novos alunos e professores. Com os seguintes campos: nome, email-institucional e senha.                           |
| RF02 | Login de usuário                  | O sistema deve permitir autenticação de alunos e professores com email e senha                                                                          |
| RF03 | Logout                            | O sistema deve permitir o encerramento da sessão                                                                                                        |
| RF04 | Recuperação de senha              | O sistema deve permitir que o usuário solicite a redefinição de senha por email, recebendo um link de recuperação.                                      |
| RF05 | Edição de perfil                  | O sistema deve permitir que usuários editem suas informações pessoais.                                                                                  |
| RF06 | Cadastro de projeto de extensão   | O sistema deve permitir que professores cadastrem projetos.                                                                                             |
| RF07 | Edição                            | O sistema deve permitir que o professor responsável edite seus próprios projetos cadastrados                                                            |
| RF08 | Desativação de projeto            | O sistema deve permitir que o professor responsável desative seus próprios projetos                                                                     |
| RF09 | Criação de processo seletivo      | O sistema deve permitir que o professor crie um processo seletivo vinculado a um projeto.                                                               |
| RF10 | Listagem de projeto               | Exibir catálogo de projetos ativos com filtros por área temática, tipo de vaga e palavras-chave.                                                        |
| RF11 | Candidatura a processo seletivo   | O sistema deve permitir que alunos se inscrevam em processos seletivos por meio de formulário, podendo anexar documentos quando solicitado.             |
| RF12 | Acompanhamento de inscrições      | O sistema deve permitir que o aluno visualize o status de todas as suas candidaturas(em análise, aprovado, rejeitado).                                  |
| RF13 | Favoritar projetos                | Alunos tem permissão para marcar projetos como favoritos para acesso rápido.                                                                            |
| RF14 | Dashboard do aluno                | Exibir para o aluno  um painel com suas inscrições ativas, projetos favoritados e notificações recentes.                                                |
| RF15 | Visualização de candidatos        | O sistema deve exibir ao professor a lista de alunos inscritos em seus processos seletivos, com acesso aos dados de cada candidato.                     |
| RF16 | Aprovação de candidatos           | O sistema deve permitir que o professor aprove candidatos por meio de um painel Kanban, movendo-os entre etapas do processo seletivo.                   |
| RF17 | Rejeição de candidatos            | O sistema deve permitir que o professor rejeite candidatos.                                                                                             |
| RF18 | Notificações                      | O sistema deve notificar alunos e professores sobre eventos relevantes: confirmação de inscrição, resultado do processo seletivo e novas oportunidades. |
| RF19 | Sugestão de novos projetos        | O sistema deve permitir que alunos sugiram ideias de novos projetos, visíveis a professores em formato de nuvem de palavras.                            |
| RF20 | Listagem de projetos do professor | Exibir ao professor um painel com todos os projetos e processos seletivos criados pro ele.                                                              |
